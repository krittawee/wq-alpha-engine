<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → cli.py seed_file path | File path argument is attacker-controlled in theory; path traversal is irrelevant for a local single-user tool |
| seeds.txt → expressions | Expression strings are user-authored; treated as untrusted input by validate.py and passed to BRAIN API |
| .env → credentials | WQ_EMAIL/WQ_PASSWORD read by wq_login.py; cli.py must not print or log these |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | 401 re-auth attempt | mitigate | 401 HTTPError is caught at the top level, printed, and causes sys.exit(1) — login() is called exactly once before the loop |
| T-05-02 | Information Disclosure | cli.py print output | mitigate | Print only expression (truncated to 40 chars), status, and numeric stats — never raw session/auth data |
| T-05-03 | Denial of Service | --workers argument | mitigate | argparse choices=[1,2,3] enforces BRAIN slot cap at the CLI boundary; no user can pass 4+ |
| T-05-04 | Information Disclosure | seeds.txt in git | accept | seeds.txt contains FastExpr expressions only — no credentials; safe to commit |
| T-05-SC | Tampering | pip installs | accept | No new packages — cli.py uses only stdlib argparse/sys/uuid/datetime and already-installed requests |
</threat_model>
