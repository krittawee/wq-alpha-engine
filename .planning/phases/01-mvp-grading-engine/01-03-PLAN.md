    - validate(conn, "rank(returns)") with rank NOT in operators → (False, "unknown operator: rank")
    - validate(conn, "rank(returns") with unbalanced parens → (False, "unbalanced parentheses")
    - validate(conn, "  ") or validate(conn, "") → (False, "empty expression")
    - validate(conn, "returns") with returns in datafields but no function call → (True, "") — bare field reference is valid
    - validate(conn, "rank(bad_field)") with rank in operators but bad_field not in datafields → (False, "unknown data field: bad_field")
    - validate(conn, "add(rank(returns), ts_mean(returns, 5))") with all tokens present → (True, "")
  </behavior>

  <action>
    Create validate.py. Import only: sqlite3, re, from typing import Optional.

    Implement validate(conn: sqlite3.Connection, expression: str) -> tuple[bool, str]:

    Step 1 — Empty check: if not expression.strip(), return (False, "empty expression").

    Step 2 — Bracket balance: if expression.count("(") != expression.count(")"), return (False, "unbalanced parentheses").

    Step 3 — Token extraction: use re.findall(r'[A-Za-z_][A-Za-z0-9_]*', expression) to get all word tokens.

    Step 4 — Operator detection: find function-call tokens by checking if the character immediately after the token in the expression is "(". Build a set of operator_tokens = {token for each token where expression contains token followed by "(" — use a regex like re.findall(r'([A-Za-z_][A-Za-z0-9_]*)\s*\(', expression) to get all called function names}.

    Step 5 — Operator validation: for each token in operator_tokens, query: conn.execute("SELECT 1 FROM operators WHERE name=? LIMIT 1", (token,)).fetchone(). If None (not found), return (False, f"unknown operator: {token}").

    Step 6 — Data field detection: all_tokens minus operator_tokens gives bare field tokens. Filter out Python keywords and numeric-only tokens using a small exclusion set: {"if","else","and","or","not","in","True","False","None"}. For each bare field token, query: conn.execute("SELECT 1 FROM datafields WHERE id=? LIMIT 1", (token,)).fetchone(). If None, return (False, f"unknown data field: {token}").

    Step 7 — Return (True, "").

    Write validate_test.py with inline test cases covering the behavior block above. Use a temporary in-memory SQLite db seeded with test rows. Run tests by executing validate_test.py directly (not pytest — keep zero new deps). The test file should be self-contained: create in-memory conn, seed operators/datafields, run assertions, print "ALL VALIDATE TESTS PASSED".

    Note: the operators table may be empty when the catalog hasn't been synced yet. In that case, all operator tokens will fail validation. This is the correct behavior — sync must run before grade.
