Implementation decisions locked on top of `999.1-SPEC.md`:

## Decisions

1. **Template generator location** ‚Äî extend `ideator.py` with a template-mode that emits a
   `{expression, slots, archetype, settings_archetype}` dict (same schema as `templates.TEMPLATES`),
   reusing ideator's existing catalog grounding + LLM call rather than a new module.

2. **Loop orchestration** ‚Äî new module `handoff.py` owns the bounded generate‚Üísweep‚Üírepeat loop
   (gen template ‚Üí validate/register ‚Üí `bruteforce.bruteforce(...)` ‚Üí check budget/cap/target).
   `/hunt --bruteforce-handoff` is a thin branch in `hunt.py` that calls `handoff.run(...)`.
   Rationale: keeps hunt.py from growing; isolates the loop concern so it's unit-testable without
   the full hunt research/mutate loop.

3. **Generator prompt grounding inputs** ‚Äî the template generator gets all four:
   - catalog summary (verified operators + datafields by dataset/type) ‚Üí slots stay valid
   - researcher thesis / archetype ‚Üí shape direction
   - failure avoid-list (past dead templates/expressions from DB) ‚Üí don't repeat crowded shapes
   - additivity hint (existing-book signal summary) ‚Üí aim for uncorrelated shapes
