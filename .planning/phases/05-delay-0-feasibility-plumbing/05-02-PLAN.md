       - coerced: bool        — True if returned_delay != requested_delay

    4. def run_probe(client, conn, requested_delay: int = 0, settings_override: dict = None) -> ProbeResult:
       - A non-throwing diagnostic helper that ALWAYS returns a ProbeResult (never raises DelayCoercedError).
       - Builds probe_settings: if settings_override is not None, use it as-is (allows Test B to pass a UI-verbatim payload without bypassing probe_delay.py). Otherwise build {**grade._BASE_SETTINGS, "delay": requested_delay}.
       - Calls grade._simulate_to_alpha(client, PROBE_EXPRESSION, settings=probe_settings).
       - Reads returned_delay: try alpha_dict["settings"]["delay"], fall back to probe_settings["delay"] if key absent or None.
       - Normalizes: returned_delay_int = int(returned_delay) if returned_delay is not None else int(probe_settings.get("delay", requested_delay))
       - Prints "[probe_delay] Probe result: requested delay={requested_delay} BRAIN returned delay={returned_delay_int}" to stderr.
       - Returns ProbeResult(returned_delay=returned_delay_int, alpha_id=alpha_dict.get("id","unknown"), requested_delay=requested_delay, coerced=(returned_delay_int != requested_delay), returned_settings=alpha_dict.get("settings",{}), settings_sent=probe_settings).
       - 401 from _simulate_to_alpha propagates unchanged.
       - conn parameter is reserved for future probe-result recording; not used in this version. Document this in the docstring.

    5. def probe_and_gate(client, conn, requested_delay: int = 0, settings_override: dict = None) -> ProbeResult:
       - The gating wrapper. Calls run_probe(client, conn, requested_delay, settings_override).
       - If result.coerced: raises DelayCoercedError(requested_delay=requested_delay, returned_delay=result.returned_delay, alpha_id=result.alpha_id, message=f"BRAIN coerced delay={requested_delay} -> delay={result.returned_delay}. Delay-0 is not working on this session. Aborting to avoid burning sim slots.")
       - If not coerced: returns result.
       - Never calls login(). 401 propagates unchanged.

    Key constraints:
    - probe_and_gate and run_probe NEVER call login().
    - Both count as 1 simulation toward the concurrency cap.
    - Do NOT call client.simulate() directly — use grade._simulate_to_alpha().
    - _simulate_to_alpha does NOT persist to DB — probe sims are ephemeral.

    Add a module-level docstring explaining: the probe-gate contract, D-04 rationale, the intentional coupling to grade._simulate_to_alpha (private function; same project; avoids duplicating retry logic), and that conn is reserved for future recording.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| probe_delay → BRAIN API | Sends one simulation request; returns delay integer from response |
| client (caller-provided) | Authenticated by caller before probe_and_gate(); probe never re-auths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Denial of Service | probe_and_gate burns a sim slot per call | accept | By design (D-04); one probe per run is acceptable cost to avoid burning dozens of slots on a coerced session. probe sims use _simulate_to_alpha and do NOT persist to alpha_kb.db |
| T-05-05 | Denial of Service | Biometric re-auth lockout (429 BIOMETRICS_THROTTLED) | mitigate | probe_and_gate and run_probe receive pre-authenticated client; zero auth calls inside either function; 401 propagates cleanly to caller who stops the run |
| T-05-06 | Tampering | grade._BASE_SETTINGS mutation (old run_delay0.py pattern) | mitigate | Task 2 retires run_delay0.py; Task 1 in Plan 01 eliminates the mutation pattern from the main grading path; probe_delay.py builds settings as a copy ({**grade._BASE_SETTINGS, "delay": requested_delay}) never mutating the module-level dict |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan |

No HIGH-severity threats. The probe is a single authenticated API call with predictable behavior; no new auth surface; no user-controlled input reaches the simulate call (PROBE_EXPRESSION is a hardcoded constant; settings_override is caller-supplied but does not affect auth or DB writes).
