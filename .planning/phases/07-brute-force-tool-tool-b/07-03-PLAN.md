    PRIVATE FUNCTION _bulk_sim_quota_aware(
      client, db_path: str, combos: list[tuple[str, dict]], run_id: str,
      quota_remaining: int, delay: int
    ) -> dict:
    Returns {"results": list[dict], "hit_401": bool}
    Opens ThreadPoolExecutor(max_workers=3). For each (expr, settings_dict) in combos:
    submit a future that calls a worker function. The worker:
      - Opens its own conn via db.init_db(db_path)
      - Calls grade.grade_one(client, conn, expr, run_id, settings=settings_dict, delay=delay)
      - Catches all non-401 exceptions as {"status": "error", "alpha_id": None, "expression": expr}
      - Closes conn in finally
      - Re-raises HTTPError(401) immediately
    After submitting all futures (or up to quota_remaining, submit in batches of 3),
    use as_completed to collect results one by one. After each result:
      - If quota_remaining <= 0: call executor.shutdown(wait=False); break
    Return accumulated results dict and hit_401 flag.

    PRIVATE FUNCTION _run_template(
      client, conn, db_path: str, template: dict, run_id: str,
      quota_remaining: int, delay: int, probe_size: int

    Returns {"n_combos", "n_validated", "n_probed", "n_simmed", "n_survivors",
             "n_additive", "probe_abandoned": bool, "hit_401": bool,

    Steps (per 07-RESEARCH.md §Architecture Diagram):
    1. expand_slots: combos_all = templates.expand_slots(conn, template). n_combos = len(combos_all).
    2. Validate gate: iterate all combos_all, call validate.validate(conn, expr) for each.
       valid_combos = combos that pass. Track failure_counts["validate_dropped"] count +
       up to 3 expression examples in examples["validate_dropped"].
    3. If valid_combos is empty: return early with probe_abandoned=True.
    4. Probe sample: sample = templates.probe_spread_sample(valid_combos,
       list(template["slots"].keys()), size=probe_size).

       run_id=run_id, delay=delay, max_workers=min(3, len(sample)), db_path=db_path).
       NOTE: grade_many IS acceptable for probe because it is bounded (max probe_size=5
       items) and quota-stop is not relevant at probe time (per 07-RESEARCH.md §Q1).
