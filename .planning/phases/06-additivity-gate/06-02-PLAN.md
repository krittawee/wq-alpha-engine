    _combined_book_corr(candidate_path: str, ref_paths: list) -> Optional[float]:
      Algorithm (from RESEARCH.md Finding 2 + Pattern 1):
        1. Load candidate JSON: dates + pnls from candidate_path. Return None on any read error.
        2. Build cand_map: dict[date_str, cumulative_pnl].
        3. For each ref_path in ref_paths: load JSON, build ref_map. Find overlap = sorted(set(ref_map) & set(cand_map)). If len(overlap) < 2: skip. Compute ref_pnl_seq = [ref_map[d] for d in overlap]; ref_daily = _pnls_to_daily_returns(ref_pnl_seq). For each (date, ret) in zip(overlap[1:], ref_daily): book_map[date] += ret. Increment refs_used.
        4. After loop: if refs_used < len(ref_paths) and refs_used > 0: print warning with count skipped (D-04).
        5. If refs_used == 0 or book_map empty: print WARNING "[additivity] zero book references available — proxy correlation is unavailable"; return None.
        6. overlap2 = sorted set of book_map keys that are also in cand_map. If len(overlap2) < 62: return None (need 61 daily returns → 62 cumulative points).
        7. cand_rets = _pnls_to_daily_returns([cand_map[d] for d in overlap2])
           book_rets = [book_map[d] for d in overlap2[1:]]   # skip first date (no predecessor in book daily series)
        8. n = min(len(cand_rets), len(book_rets)). If n < 60: return None.
        9. return _pearson(cand_rets[:n], book_rets[:n])

    rank_by_proxy(candidates: list, conn, margin: float = PROXY_MARGIN) -> list[AdditivityResult]:
      candidates: list of dicts with keys alpha_id (str) and pnl_path (Optional[str]).
      1. ref_paths = selfcorr.get_book_pnl_paths(conn). If empty: print WARNING "[additivity] book PnL reference set is empty — proxy rank unavailable; all candidates pass through".

      3. For each candidate:
         a. If pnl_path is None or pnl_path missing: create AdditivityResult(alpha_id, pnl_path=None, combined_corr=None, max_pairwise_corr=None, proxy_drop=False, skipped=True).
         b. Else: combined_corr = _combined_book_corr(pnl_path, ref_paths). max_pairwise_corr = selfcorr.max_pearson(pnl_path, ref_paths) (or 0.0 if ref_paths empty). proxy_drop = (combined_corr is not None and limit is not None and combined_corr > limit + margin). Create AdditivityResult(alpha_id, pnl_path, combined_corr, max_pairwise_corr, proxy_drop, skipped=False).
      4. Sort: items with combined_corr is not None → sort ascending by combined_corr. Items with combined_corr is None (skipped) → append after.
      5. Return sorted list.

    confirm_additive(client, alpha_id: str, conn, timeout: int = 300, interval: int = 15) -> AdditivityResult:

      4. sc = corr_checks.get("SELF_CORRELATION", {}). pc = corr_checks.get("PROD_CORRELATION", {}).
      5. self_result = sc.get("result"). self_value = sc.get("value"). self_limit = sc.get("limit")  # live limit, not DB.
         prod_value = pc.get("value"). prod_result = pc.get("result").
      6. If self_result is None (SELF_CORRELATION absent from response): additive=None (inconclusive).
         Elif self_result == "PASS": additive=True.
         Else: additive=False.
      7. Return AdditivityResult(alpha_id=alpha_id, pnl_path=pnl_path, combined_corr=None, max_pairwise_corr=None, proxy_drop=False, skipped=False, brain_self_corr=self_value, brain_self_corr_result=self_result, brain_self_corr_limit=self_limit, brain_prod_corr=prod_value, brain_prod_corr_result=prod_result, additive=additive).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BRAIN /check response → confirm_additive | JSON response is untrusted; c["limit"] may be absent, wrong type, or missing SELF_CORRELATION entirely |
| Local PnL JSON files → _combined_book_corr | Files from pnl_cache/ written by fetch_and_cache_pnl from BRAIN response; treated as trusted but may be malformed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Tampering | confirm_additive: malformed /check response (missing "limit" key) | mitigate | sc.get("limit") returns None gracefully; brain_self_corr_limit=None in result; additive verdict still set from sc.get("result"); never raises on missing key |
| T-06-05 | Tampering | confirm_additive: SELF_CORRELATION absent from response | mitigate | corr_checks.get("SELF_CORRELATION", {}) → self_result=None → additive=None (inconclusive, not True/False); caller in hunt.py must treat None as non-additive |
| T-06-06 | Information Disclosure | AdditivityResult.brain_self_corr_limit exposes BRAIN's live limit | accept | Limit is already in DB from grading; no new disclosure; used only internally |
| T-06-07 | Denial of Service | poll_correlation timeout (300s default per finalist) | accept | CONFIRM_LIMIT=3 caps finalists; worst-case 900s; non-blocking to sim pool |
| T-06-08 | Elevation of Privilege | hardcoded 0.7 threshold bypassing BRAIN's real limit | mitigate | confirm_additive reads c["limit"] exclusively; PROXY_MARGIN applied to get_selfcorr_limit(conn) from DB (stale-acceptable for pre-filter only); no literal 0.7 in additivity.py |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No new packages; stdlib + project modules only |
